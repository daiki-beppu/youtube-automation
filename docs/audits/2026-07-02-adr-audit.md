# ADR 監査 2026-07-02

親 issue: #1413。`docs/adr/` 全 20 件の監査。構造面（番号・Status・supersede リンク・グロッサリ整合）と内容面（各決定と実装の突合）を実施した。

## 方法

- **構造監査**: 番号重複・Status 欠落・相互リンク・CONTEXT.md との矛盾を機械的に洗い出し、本監査 PR で修正

  - TS rewrite 系 ADR（0001〜0008、0009、0015、0017、0018）: `origin/feat/ts-rewrite` ブランチ

- TS rewrite は生きている決定として扱う（epic #727 OPEN、ADR-0015 が 2026-06-25 に計画を再確認）

### Status 欠落 17/20 → 全 20 件に統一追記

語彙は `accepted (日付)` / `superseded by ADR-NNNN` / `deprecated`。日付は初回マージ日（git log から機械的に採取）。TS 系 ADR には「実装は `feat/ts-rewrite`、cutover #790 で main へ反映予定」を明記し、「accepted だが main 未反映」という中間状態を可視化した。

### supersede の片方向リンク → 解消済みを確認

ADR-0007 → 0001 §Identity の supersede は、0001 本文冒頭に既に注記があった（構造上の欠落は Status 欄のみ。追記済み）。

### グロッサリ矛盾: first-party vs 外部ユーザー

CONTEXT.md「第三者 consumer は存在しない」と ADR-0015「外部ユーザー数十人規模」が矛盾していた。**外部ユーザー実在が正**と確認。CONTEXT.md の first-party 定義を修正し **external user** を新語登録。ADR-0007（rebrand）は前提が変化したが、移行コストは ADR-0015 §移行戦略で吸収するため**決定維持**（0007 Status に注記）。

3. **ADR-0015 告知義務の履行** (#1416) — 移行告知 + 移行ガイドの公開（期限 2026-07 頭、到来済み）
4. **`@tayk/*` scope 改名** — 0006/0007 の部分乖離。lock point は最初の alpha publish 前
